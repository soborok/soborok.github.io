Enumerable Method, map

November 1, 2014

Map the elements

As its original meaning, map method draws data of something. map method is often used for array/hash and it draws elements of array/hash.
It doesn't stop at drawing elements in Ruby. According to what write in block you can show maniplulated elements. And here show means "return". Unless you write print code it won't be printed on the screen. Let's see simple example.

my_array = [1, 2, 3, 4]
my_array.map { |element| p element}

ON THE SCREEN>
1
2
3
4

In the block p method is written and map method prints the element one by one. You can do more in the block.

my_array = [1, 2, 3, 4]
my_array.map { |element| element ** 2 }

RETURNED VALUE>
[1, 4, 9, 16]

In the block element is multiplied by itself. It doesn't have print command in the block so nothing will be displayed on the screen. But map method would return new value which is [1, 4, 9, 16]. But the new value [1, 4, 9, 16] is not my_array.

Keep original as original

map method draw, manipulate and return value of elements but it doesn't change the original value. Let's check by printing my_array after map method.

my_array = [1, 2, 3, 4]
my_array.map { |element| element ** 2 }
p array

ON THE SCREEN>
[1, 2, 3, 4]

map method kept my_array as it was. map method is non-destructive method. By adding ! to map, map! method becomes destructive method and it would change elements.
