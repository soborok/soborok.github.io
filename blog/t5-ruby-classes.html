Class is alive

November 9, 2014

Class, an active object

Class might be vague thing to explain. It has some elements inside and it can do some work with the elements. It will store values to their elements and will return some output when we ask them. Class itself doesn't do anything but it creates an instance and makes it alive and functional.
I like to use real life object to explain programming concept. And I am going to use coffee station to explain about class.
Coffee station has everything to make latte. There is coffee beans, milk and syrup. These should be accessbile to make latee so they are created as instance variables. In coffee station you can make latte and you can check how much coffee, milk you have. You will have thsese barista's work as instance methods.
This can be described in Ruby class as below

class CoffeeStation

  def initialize(coffee, milk, syrup)
    #this sets amount of coffee, milk, and syrup.
    @coffee = coffee
    @milk = milk
    @syrup = syrup
  end

  def make_latte(number_of_drink, number_of_shots, syrup_amount)
    get cup ready
    make esspresso and update @coffee by subtracting number_of_shots
    froth milk and update @milk by subtracting number_of_drink
    add syrup and update @syrup by subtracting syrup_amount
  end

  def show_what_I_have
    puts "You can make #{@coffee} esspresso shots and #{@milk} cups of warm milk. And you have #{@syrup} pumps of syrup."
  end

end

I want a coffee station at home. I will create a CoffeeStation instance 'myCoffeeStation' with 1 whole bag of coffee bean which can make 50 esspresso shots, 1 carton of milk which can makes 20 cups of drink and 1 bottole of vanila syrup which can provide 100 pumps of syrup.
And I am going to make a double shot vanila latte at my coffee station. 'myCoffeeStation' instance will call make_latte method.

myCoffeeStation = CoffeeStation.new(50, 20, 100)
myCoffeeStation.make_latte(1 latte, 2 shots, 2 pumps of vanila syrup)

Uhm~ I made vanila latte and it tastes good. But it is bit strong and not sweet enough for my sister. So I will make single shot latte with 3 pumps of vanila syrup for her. I will call make_latte method again but this time parameter will have different numbers.

myCoffeeStation.make_latte(1 latte, 1 shots, 3 pumps of vanila syrup)

Good, she likes it.
Now I want to see how much I have in my coffee station and I will call show_what_I_have method.

myCoffeeStation.show_what_I_have

And this will display a message

You can make 47 esspresso shots and 18 cups of warm milk. And you have 95 pumps of syrup.

CoffeeStation is a simple class and doesn't explain all about class. But it would give a basic idea what class is, how to make an instance, what data the instance has and how to call methods.
You can create an instance for your mocha drink with CoffeeStation class. And if I have more time I can extend CoffeeStation class to serve more than one kind of drink.
