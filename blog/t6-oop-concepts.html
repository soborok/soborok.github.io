Variable Scope

November 14, 2014

variables - local, instance, class and global

To know variable scope of Ruby, I need to see how a class looks like.
A class is defined starting with 'class' keyword and terminatined by 'end' keyword. Inside the difinition, it has methods, variables and calls. I can imagine what different types of variables exists by looking at they are placed in class definition.

Let's get familiar how they are written.

local_variable            always found in my code and it works in the method
@instance_variable    seen in initialize method of a class
@@class_variable      need double consideration to use this class variable
$global_variable        looks global cause it has $ sign!

Local variable works where it is defined. It cannot be accessed from another class or method. When the mothod is ovr the local variable is no longer alive.

Instance variable is available for instances of a class. I'd like to say intance variable is owned by an instance. This mean each instance will have the same name instance variable but each stores different value.

Class variable is available across instances. This means the class variable can be accessed by instance A and instance B. When instance A changes the class variable, instance B will have the changed class variable.

Global variable is available accross classes. It can be accessed out of the method or class.

Let's see an example.

class Student
  @@number_of_students = 0
  def initialize(input_name, input_city)
    @name = input_name
    @city = input_city
    @@number_of_students += 1
  end

  def number_of_students
    @@number_of_students
  end

  def display
    "#{@name} is from #{@city} and attending #{$location} cohort."
  end
end

$location = "San Francisco"

student_jen = Student.new("Jen
Chicago")
puts student_jen.number_of_students
puts student_jen.display

student_andrew = Student.new("Andrew
New York")
puts student_andrew.number_of_students
puts student_andrew.display

puts student_jen.number_of_students

The output will be

1
Jen is from Chicago and attending San Francisco cohort.
2
Andrew is from New York and attending San Francisco cohort.
2

$location is a global variable. It is used in display method of Student. But it wasn't passed into the methode unlikely other variables.

@@number_of_students is a class variable of class "Student". An instance of "Student" class can access @@number_of_students. When student_jen instance was created @@number_of_students increased by 1 and student_jen printed it out. When student_andrew instance was created @@number_of_students increased by 1 again and now @@number_of_students is 2. student_jen and student_andrew printed out the same number which is "2".

@name and @city are instance variables of "Student" class. student_jen has @name and @city. student_andrew has @name and @city, too. Instance variable names are the same as @name and @city but they store different values. This is because each instance owns the instance variables.

input_name and input_city are locali variables in initialize method. They store the values pass the values into instance variables which are @name and @city. input_name and input_city cannot be accessed in another method.

You wouldn't want every object to access every variable and mess up with them. You don't want Andrew to change Jen's name.
You would want to access a variable where you need it. Jen wants to know how many student is attending San Francisco cohort.
You wouldn't want variables to take memory unnecessarily for all methods. input_name and input_city are stored in instance variables. They don't need to be available for display or other methods.

I drew a simple chart; global > class > instance > local
