Ruby Hashes vs. JavaScript Objects

November 23 , 2014

Object oriented with different approach

Learning 2 languages at the same time is confusing. Ruby and JavaScript are both object-oriented languages but they seem to have different approch. It will need time and practice to understand and write code flawlessly of the 2 similar but different languages. This week I got a chance to compare one of their concept.

Hash of Ruby and object of JavaScript look very similar. They are defined with {} and both store a pair of key-value or property-value. The values don't have to be the same data type. Both can store strings, numbers and arrays.
JavaScript object can store one more thing which is function. This is what a class has in Ruby called method. A class has variables like string variable, number variable, boolean variable, array variable or hash variables and methods like read method, write method or manipulate(function) method. Ruby is class-based language and JavaScript is not and it doesn't have classes.
Interestingly JavaScript object looks like hash, but behaves like a class. It will be easy to look at example.

puts("I am #{my_dog["name"]}.")
var my_dog = {
  name: "Luna",
  age: 5,
  hungry: true,
  speak: function() {
    console.log('I am ' + this.name + '.') }
};

my_dog.speak();


Class vs. Object
class Pets
  attr_reader :name, :age, :hungry
  def initialize(name, age, hungry)
    @name = name
    @age = age
    @hungry = hungry
  end

  def speak
    puts "I am #{@name}."
  end
end

luna = Pets.new("Luna", 5, true)
luna.speak
var my_dog = {
  name: "Luna",

};

my_dog.speak();

In JavaScript I have to create another object for my cat. But in Ruby, I will just create a Pets instance and I don't have to write the properties again.
